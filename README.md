# Lartek — API сумісних моделей

Серверна частина для блоку «Сумісні моделі»: база сумісності + пошук.
Повного списку моделей у сторінці немає — фронтенд тягне дані звідси (пошук + прев'ю),
тож список захищений від копіювання й сторінки лишаються легкими.

## Файли
- `server.js` — API (пошук, прев'ю, імпорт), CORS, обмеження частоти
- `db.js` — підключення до Postgres + нормалізація коду моделі
- `schema.sql` — таблиця `compatibility` + індекси (вкл. триграмний для швидкого пошуку)
- `import-csv.js` — масовий імпорт із CSV (`sku,brand,model`)
- `sample.csv` — приклад даних (тестовий товар `DEMO123`)
- `.env.example` — приклад змінних оточення

## Деплой на Railway
1. Залий цю папку в GitHub-репозиторій (або `railway up` із папки).
2. На Railway: **New Project → Deploy from GitHub repo** (або додай сервіс у проєкт `pacific-victory`).
3. Додай базу: **New → Database → PostgreSQL**. Railway створить змінну `DATABASE_URL`.
   - Якщо сервіс її не бачить автоматично — у Variables сервісу додай `DATABASE_URL` = `${{Postgres.DATABASE_URL}}`.
4. У Variables сервісу додай свій секрет: `IMPORT_KEY=<довгий_випадковий_рядок>`.
5. **Deploy.** При старті сервер сам створює таблицю (виконує `schema.sql`).
6. **Settings → Networking → Generate Domain** — отримаєш публічний URL API (далі `API_URL`).

## Завантаження даних

**Найпростіше — кнопкою в браузері (без терміналу):** відкрий `https://<домен-API>/admin`,
встав ключ `IMPORT_KEY`, за бажанням впиши артикул (порожньо = весь сайт) і натисни
«Імпортувати з фіду». Дані з описів фіду самі потраплять у базу. Весь сайт займає ~30–60 сек.

**Прямо з фіду Horoshop (термінал):** таблиці «Сумісний із моделями» вже є в описах
товарів у YML-фіді. `import-feed.js` витягує їх і заливає в БД — CSV готувати не треба:
```
# пілот — лише один товар, без запису (перевірка парсингу):
node import-feed.js --sku=0873 --dry-run
# пілот — залити один товар:
DATABASE_URL="postgres://...railway..." node import-feed.js --sku=0873
# увесь сайт:
DATABASE_URL="postgres://...railway..." node import-feed.js
# перезалити (спершу чистить старі моделі кожного товару):
DATABASE_URL="postgres://...railway..." node import-feed.js --replace
```
Парсер: бренд — слово з великої літери, модель — токен із цифрою; на prose-тексті після
таблиці розбір зупиняється. Перед повним заливанням сайту прогони `--dry-run` і звір кількість.

**Масово з CSV:** підготуй CSV із колонками `sku,brand,model` і запусти локально,
підставивши публічний `DATABASE_URL` із Railway (вкладка Postgres → Connect → Public Network):
```
DATABASE_URL="postgres://...railway..." node import-csv.js data.csv
```
**По одному товару (через API):**
```
curl -X POST "$API_URL/api/import" \
  -H "Content-Type: application/json" \
  -H "X-Import-Key: <твій IMPORT_KEY>" \
  -d '{"sku":"АРТИКУЛ","replace":true,"models":[{"brand":"Bosch","model":"SMV68IX00D/01"}]}'
```
`replace:true` спершу чистить старі моделі цього товару; прибери його, щоб лише додавати.

## Ендпойнти
- `GET /health` → `ok`
- `GET /api/brands?sku=АРТИКУЛ` → `{ "total":N, "brands":[{brand,count}] }` (для випадайки брендів)
- `GET /api/models?sku=АРТИКУЛ&q=SMV68` → `{ "items":[{brand,model}] }` (швидкий пошук)
  - `&brand=Bosch` — перегляд усіх моделей бренду (лівий список; стеля `BROWSE_CAP`)
  - `&brand=Bosch&q=SMV` — пошук у межах бренду
  - закоротко: `{ "tooShort":true, "min":3 }`
  - забагато збігів: `{ "tooMany":true, "cap":40 }`
- `GET /api/preview?sku=АРТИКУЛ` → `{ "total":N, "items":[...] }` (невеликий список + кількість)
- `POST /api/import` (заголовок `X-Import-Key`) → `{ "sku":..., "processed":N }`
- `POST /api/import-feed` (заголовок `X-Import-Key`, тіло `{ "sku"?, "replace"? }`) → наповнення з фіду
- `GET /admin` → сторінка-кнопка для наповнення з фіду (без терміналу)
- `GET /api/export` (заголовок `X-Import-Key`) → `{ "count":N, "items":[{ "sku", "models":[...] }] }`
  - уся сумісність, згрупована по товарах; для індексатора Meili (приховане пошукове поле).
  - `?sku=АРТИКУЛ` — тільки один товар. Захищено ключем, щоб повний список не був публічним.

## Під'єднання компонента
Готовий блок для вкладки «Сумісні моделі» — `embed/compat-widget.html`. Самодостатній
(HTML+CSS+JS, без залежностей). Вставляється замість повної статичної таблиці й показує
лише **кількість** сумісних моделей + поле пошуку; повний список не рендериться.

Перед вставкою у блоці задай два атрибути:
- `data-api` — публічний домен твого `models-api` (Railway → Generate Domain);
- `data-sku` — артикул товару (пілот: `0873`; для сайту — шаблонна змінна Horoshop з артикулом).

Він звертається до:
- прев'ю при завантаженні: `GET $API_URL/api/preview?sku=<АРТИКУЛ>`
- пошук при вводі: `GET $API_URL/api/models?sku=<АРТИКУЛ>&q=<ввід>`

Нормалізація пошуку у фронтенді збігається з серверною (`db.js` → `norm`): великі
літери, лише `A-Z0-9`. Тому «ro-5295 ea» і «RO5295EA» дають однаковий результат, а пошук
лишається точним, літера-в-літеру (підрядок), без фаззі/одруківок.

## Пошук по сайту за моделлю (щоб не втратити після приховування списку)
Коли список прибрано з описів, загальний пошук Meilisearch перестане знаходити товар за
номером техніки. Щоб цього не сталося, індексатор (`meili-indexer`) тягне моделі з
`GET /api/export` і кладе у **приховане** пошукове поле (`models`): воно шукається, але не
показується й не віддається в браузер. Індексатору потрібні змінні `MODELS_API_URL` і
`MODELS_API_KEY` (= цей `IMPORT_KEY`).

## Порядок впровадження (пілот, без ризику)
1. Задеплой цю версію `models-api` (з `/api/export`).
2. Наповни БД пілотним товаром: відкрий `https://<домен-API>/admin`, впиши артикул `0873`,
   натисни «Імпортувати» (або `node import-feed.js --sku=0873`).
3. Задеплой `meili-indexer` зі змінними `MODELS_API_URL`/`MODELS_API_KEY`, запусти індексацію —
   переконайся, що пошук по сайту знаходить товар за моделлю (приховане поле).
4. Встав `embed/compat-widget.html` у вкладку «Сумісні моделі» (з `data-api`/`data-sku`), перевір.
5. Аж тепер прибери таблицю з опису товару в Horoshop. Пошук уже тримається на прихованому полі.

## Налаштування (Variables)
- `MIN_CHARS` — мінімум символів для пошуку (за замовч. 3)
- `RESULT_CAP` — стеля видачі, далі «уточніть» (за замовч. 40)
- `PREVIEW_LIMIT` — розмір прев'ю (за замовч. 12)

Домени, з яких дозволено звертатись до API, задані в `server.js` (масив `ALLOWED`):
`lartek.com.ua` і `komplektom.com.ua` (з `www`). Додай інші за потреби.
